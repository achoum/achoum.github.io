Operator index

Operator Definition
AUTODATASET Define the input and output files/directory on which to apply the script. This command is generaly used to replace input/output specification in the command line.
active Compute the presence or the absence of signal in the tailing moving window. If a signal is present, the operator returns 1, otherwise the operator returns 0.
autoTest Run a statistical test between the current value, and past values with the same time distance to the specified landmarks.
beep Emit a beep (computer noise) every time an event is received.
beginGroup Begins a group of operations. Grouping operation does not impact processing. Grouping operation defines cluster of operations that will be grouped together for the script graphical representation. The group should be closed with the "endGroup" operator.
calendar Generate hour/day/month change point events and status. The time is interpreted as unix time i.e. number of second since the 1 Jan. 1970.
corr Compute the value and the time shirt that maximise the correlation between two signals.
count Count the number of signal input.
cusum Moving CuSum (Cumulative Sum).
delay Delay a signal i.e. create an artificial lag. A delay is positive. For "negative delay", look at the "echoPast" operator.
derivative Compute the numerical derivative between the current and previous values i.e. the difference between the last two values normalized by the time interval. If the signal is nosy, it is recommanded to apply a "tma" (triangular mobing average) before this operator.
div Divide two signal values.
echo Repeats a signal. Be carefull with infinite loops when using direct names or regular expression catching. To avoid any looping, it is recommanded for the first line of the script to catch all the signal into a variable (i.e. $ALL = echo #.*), and then to use the "filter" operator on the $ALL variable.
echoPast Similar to "echo", be allow for the time to be decremented i.e. send in the past. Obviously, echoPast does not work in online mode.
ema Apply a Tailing Exponential Moving Average with a fixed window length.
endGroup Close of group. See operation "beginGroup" for more details.
eq Return the result of a Polish Notation equation on the value of several signals. Dynamic variables are "time", "value", "arg1",..., "arg4".
ewma Exponential weighted moving average control chart operator.
fft Moving Fast Fourier Transform. The resulting spectum is grouped into bins. Each bin is emited as an individual output.
filter Filter the signal from a variable using a regular expression.
finish Generates the event "finish" at the end of the script execution.
global_normalize Normalize the signal in mean and standart deviation. Since the normalization requiere to know all the signal values, this operator cannot be used in online mode. For normalization in online mode, you can use the operators "normalize" or "normalizeFixedTime".
keyboard Emit an event when a keyboard key is pressed. Only work in online mode.
layer Suppose a set of thresholds evenly spaced between "min" and "max". Generates an event whenever the input signal crosses up or down any of the thresholds.
max Maximum over a tailling window.
min Minimum over a tailling window.
multiTests Combine several statistical tests p-values. The input should be the -log of the p-values.
normalize Normalize the signal by mean and mean+variance computed on a tailing window.
normalizeFixedTime Normalize the signal by a mean and mean+variance of the signal. The mean and variance are computed in the indicated time window. A signal is generated when at least 5 points are met.
passIf Repeats a signal if the result of the equation applied on a signal is greater or equal to 1. Look at the "eq" operator for more details. Allowed terms are "time", "value", "arg1",..., "arg4".
passIfFast Repeats a signal if all constraints are satisfied. This operator is significantly faster than "passIf".
peak Detect and characterize peaks (i.e. value of the second order redivative when the first order derivative of the signal crosses the value 0).
print Print the content of a variable. If exectuted in static mode, print will print statistics on the data.
range Moving range (i.e. maximum value - mininum value ) of the signal.
rename Change the symbol of a signal. By default, if several different signals are provided, and error is generated. If keepAll:true and if several signals are provided, all the signals are merged into a same signal. If keepOnlyFirst:true and if several signals are provided, only the first meet signal is renamed and the others are ignored.
renameRegexp Change the symbol of a signal or a list of signal using a regular expression replacement.
report_histIntersectEventScalar Count the number of instance of a symbol according to the value of a scalar. You can for example look for the number of one type of event according by the hour of the day (if the later is represented as a scalar value).
report_histIntersectEventState Count the number of instance of a symbol according to the value of a state. You can for example look for the number of one type of event according by the hour of the day (if the later is represented as a scalar state).
rest Repeats a signal if its value is greater than any of its previous value in a tailling window.
return Return a set of signal to the script called. Return is mainly used with Event Script API.
sample Sample a signal according to a trigger signal.
save Save the signal to .evt file. Each received signal is immedialy saved and released from memory.
saveBufferedBin Save the signal to a .bin file. All the events are stored in a buffer until the end of the script execution.
saveBufferedCsv Save the signal to a .csv file. The dataset is stored in a buffer until the end of the execution. If the dataset is too largue to fit in memory, use the "savecsv" or "save" operators.
saveCsv Save the signal to a file in the CSV format. Each received signal is immedialy saved and released from memory. However, unlike the "saveBufferedCsv" operator, you need to indicate the output collumns. If the collumn names are not specified, they will be infered when writting the first line of the file. In this case, all signals should be available when writing the first line.
saveSnapshot Save a snapshot of the signal either at the end of the script execution, or when a trigger event is received.
sd Moving standard deviation.
segment Extract pattern features from sequenced signal.
set Assign a value to a variable.
shewhart_univ Apply a Shewhart individuals control chart.
sinceLast Time difference between the current and the last input signal update.
skip Subsample a signal by only repeating update which are far enough from the direct previour repeated update.
sma Applies a tailing simple moving average with a time fixed window length.
sub Substract the value of two signals.
sum Sum the value of a signal updates over a fixed window length.
tcp Create a tcp client connection to a tcp server. The communication is done with the .evt format.
tcpserver Create a tcp server. The communication is done with the .evt format.
tick Generate a tick signal at regular interval.
tma Apply a Tailing Triangular Moving Average with a fixed window length.
windowfeatures Compute several common tailling window features. Computed features are: mean (equivalent to SMA [simple moving average]), min, max, median, standart deviation, range, and 5%-95% range.
zigzag Compute zigzag pattern detection.

AUTODATASET
Define the input and output files/directory on which to apply the script. This command is generaly used to replace input/output specification in the command line.
Named options
extension [optionnal] Output extension in the case the output argument is a directory. Default value: "bin".
input Input file path or input directory path. Several files can be provided if separated by a ";". Supported formats are .evt, .csv, .bin and .sevt.
output Output file/directory path.
Example
AUTODATASET input:"input.evt" output:"output.csv"
AUTODATASET input:"input1.evt;input2.evt" output:"output.csv"
AUTODATASET input:"input_directory" output:"output_directory" extension:bin
